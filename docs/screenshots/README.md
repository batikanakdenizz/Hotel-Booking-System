# Screenshots

Drop the demo screenshots referenced in the project [`README.md`](../../README.md)
into this directory. Suggested resolution: **1600 × 900** for full-page captures,
**800 × 500** for component-level shots. PNG or WebP both work.

| File | What to capture | Where it shows up |
|---|---|---|
| `01-home.png` | Full landing page hero + search bar + popular destinations grid | Top of README |
| `02-ai-chat.gif` _(optional)_ | Floating chat with a prompt + LLM response | "Live demo" section |
| `03-admin.png` | `/admin/hotels` page with the New hotel form open | "Hotel admin" feature |
| `04-search.png` | Search results page with the Leaflet map sidebar populated | "Hotel search" feature |
| `05-my-bookings.png` | `/my-bookings` after a fresh booking is confirmed | "Transactional booking" feature |
| `06-comments-chart.png` | Hotel detail page showing the 5-dim Recharts bar chart + a comment | "Reviews" feature |
| `07-ai-chat.png` | Chat widget with a real hotel-list response visible | "AI agent" feature |
| `08-architecture.png` _(optional)_ | Higher-res or whiteboard variant of the Mermaid diagram | "Architecture" section |
| `09-render.png` | Render dashboard listing all 7 services Live | "Production deployment" |
| `10-vercel.png` | Vercel project deployments list with "Ready" on the latest commit | "Production deployment" |

Filenames are referenced verbatim from the README — if you rename one,
update the `<img src="docs/screenshots/...">` tag in the README to match.
